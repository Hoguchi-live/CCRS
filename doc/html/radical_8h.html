void fq_nth_root_trick ( fq_t  rop,
fq_t  op,
fmpz_t  l,
const fq_ctx_t  F 
)

Extract n-th root of op using the following trick: we always have op ^ (p + 1) = op ^ 2 and assuming p + 1 = 2*l*k for some natural number k, we get (op ^ e)^l = +/- op for e = (p + 1) / (2 * l). We get rid of the +/- sign by testing via fast exponentiation. A NAS condition for this to work is p + 1 = 0 % 2 * l. This is about 1000 times faster than bruteforcing roots of X^l - op.
