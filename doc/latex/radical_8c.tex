{\footnotesize\ttfamily void fq\+\_\+nth\+\_\+root\+\_\+trick (\begin{DoxyParamCaption}\item[{fq\+\_\+t}]{rop,  }\item[{fq\+\_\+t}]{op,  }\item[{fmpz\+\_\+t}]{l,  }\item[{const fq\+\_\+ctx\+\_\+t}]{F }\end{DoxyParamCaption})}

Extract n-\/th root of op using the following trick\+: we always have op $^\wedge$ (p + 1) = op $^\wedge$ 2 and assuming p + 1 = 2$\ast$l$\ast$k for some natural number k, we get (op $^\wedge$ e)$^\wedge$l = +/-\/ op for e = (p + 1) / (2 $\ast$ l). We get rid of the +/-\/ sign by testing via fast exponentiation. A NAS condition for this to work is p + 1 = 0 \% 2 $\ast$ l. This is about 1000 times faster than bruteforcing roots of X$^\wedge$l -\/ op. \mbox{\Hypertarget{radical_8c_ab90b26fac989284288d235295587ccf2}\label{radical_8c_ab90b26fac989284288d235295587ccf2}}
